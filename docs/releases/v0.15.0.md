# Release Notes: v0.15.0

**Release Date:** 2026-02-28
**Type:** Architecture Refactor (Phase 1)

## Summary

Phase 1 of the execution module re-architecture. Extracted god-file logic from `runner.ts` into focused sub-modules within new `lifecycle/` and `escalation/` subdirectories.

## Architecture Changes

### New Directory Structure

```
src/execution/
  lifecycle/
    run-lifecycle.ts          — setup/teardown orchestration
    acceptance-loop.ts        — acceptance retry loop
    story-hooks.ts            — reporter notification helpers
    index.ts                  — barrel exports
  escalation/
    escalation.ts             — tier escalation helpers (moved from root)
    tier-escalation.ts        — pre-iteration checks + tier handling
    index.ts                  — barrel exports
  runner.ts                   — thin orchestrator (reduced from 1,638 to 1,428 lines)
```

### Extracted Modules

1. **`lifecycle/story-hooks.ts`** (new)
   - `emitStoryComplete()` — centralized reporter notification
   - Eliminates 4+ duplicated reporter emission blocks (32 lines saved)

2. **`escalation/tier-escalation.ts`** (new)
   - `resolveMaxAttemptsOutcome()` — pause vs fail decision logic
   - `preIterationTierCheck()` — pre-iteration budget validation
   - `handleTierEscalation()` — pipeline escalation handler (203 lines extracted)

3. **`lifecycle/run-lifecycle.ts`** (existing, now in lifecycle/)
   - Moved from `src/execution/run-lifecycle.ts` to `lifecycle/` subdir

4. **`lifecycle/acceptance-loop.ts`** (existing, now in lifecycle/)
   - Moved from `src/execution/acceptance-loop.ts` to `lifecycle/` subdir

5. **`escalation/escalation.ts`** (existing, now in escalation/)
   - Moved from `src/execution/escalation.ts` to `escalation/` subdir
