## PRD Format

User stories are defined in `nax/features/<name>/prd.json`:

```json
{
  "feature": "user-auth",
  "userStories": [
    {
      "id": "US-001",
      "title": "Add login endpoint",
      "description": "POST /auth/login with email/password",
      "acceptanceCriteria": [
        "Returns JWT on success",
        "Returns 401 on invalid credentials"
      ],
      "complexity": "medium",
      "tags": ["auth", "security"],
      "status": "pending"
    }
  ]
}
```

> **Note:** Use `"status": "passed"` (not `"done"`) to manually mark a story complete.

---

### Field Reference

| Field | Type | Required | Description |
|:------|:-----|:--------|:------------|
| `feature` | `string` | Yes | Feature name (should match folder name) |
| `userStories` | `array` | Yes | Array of user story objects |
| `id` | `string` | Yes | Unique story identifier (e.g., "US-001") |
| `title` | `string` | Yes | Short story title |
| `description` | `string` | No | Detailed description of the story |
| `acceptanceCriteria` | `array` | Yes | List of criteria that must be met |
| `complexity` | `string` | No | **Legacy.** Use `routing.complexity` instead |
| `tags` | `array` | No | Tags for keyword routing and classification |
| `status` | `string` | Yes | Story status: `pending`, `passed`, `failed` |
| `workdir` | `string` | No | Override working directory for monorepos |
| `dependencies` | `array` | No | Array of story IDs this story depends on |
| `routing` | `object` | No | Routing decisions — **always generated by `nax plan`** |
| `routing.complexity` | `string` | No | `simple`, `medium`, `complex`, `expert` |
| `routing.testStrategy` | `string` | No | `no-test`, `tdd-simple`, `three-session-tdd-lite`, `three-session-tdd`, `test-after` |
| `routing.reasoning` | `string` | No | Brief classification rationale |
| `routing.noTestJustification` | `string` | No | Required when `testStrategy` is `no-test` |

> **PRD routing always wins.** When `routing.complexity` and `routing.testStrategy` are present (which `nax plan` always generates), runtime routing (keyword/LLM/plugin) is skipped entirely. To change routing for a story, edit these fields directly in `prd.json` before running `nax run`.
