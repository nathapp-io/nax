## Summary

Two reporter plugins that auto-create pull requests (GitHub) or merge requests (GitLab) when a nax run completes successfully. Ships as separate npm packages implementing the `IPostRunAction` interface.

## Motivation

After `nax run` completes a feature, users must manually create a PR/MR ‚Äî write a description, link to the feature, summarize changes. This is tedious and breaks the automated flow. A post-run action plugin can handle this automatically.

## Design Principles

1. **Plugin, not core** ‚Äî PR creation is provider-specific and opinionated; keep nax core lean
2. **Two packages** ‚Äî `@nathapp/nax-plugin-github-pr` and `@nathapp/nax-plugin-gitlab-mr` ‚Äî users install only what they need
3. **Implements `IPostRunAction`** ‚Äî uses the new extension point from PLUGIN-001 (not `IReporter`)
4. **Graceful degradation** ‚Äî if `gh`/`glab` CLI is missing or auth fails, log a warning and skip (never block the run)
5. **No LLM calls** ‚Äî PR body is built from PRD + git diff, no token spend

## Architecture

```
Runner.run() finally block
  ‚îú‚îÄ‚îÄ reporters.onRunEnd(event)          ‚Üê observability (existing)
  ‚îú‚îÄ‚îÄ postRunActions loop                ‚Üê from PLUGIN-001
  ‚îÇ   ‚îú‚îÄ‚îÄ action.shouldRun(context)
  ‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ false ‚Üí skip (stories failed, PR exists, CLI missing)
  ‚îÇ   ‚îÇ   ‚îî‚îÄ‚îÄ true ‚Üí action.execute(context)
  ‚îÇ   ‚îÇ       ‚îú‚îÄ‚îÄ success ‚Üí log PR/MR URL
  ‚îÇ   ‚îÇ       ‚îú‚îÄ‚îÄ skipped ‚Üí log reason
  ‚îÇ   ‚îÇ       ‚îî‚îÄ‚îÄ failed ‚Üí log warning, continue
  ‚îî‚îÄ‚îÄ registry.teardownAll()
```

## Package: `@nathapp/nax-plugin-github-pr`

### Config

```json
{
  "plugins": [
    {
      "module": "@nathapp/nax-plugin-github-pr",
      "config": {
        "baseBranch": "main",
        "draft": true,
        "labels": ["nax-generated"],
        "assignee": "@me",
        "template": "default"
      }
    }
  ]
}
```

| Field | Type | Default | Description |
|:------|:-----|:--------|:------------|
| `baseBranch` | string | `"main"` | Target branch for PR |
| `draft` | boolean | `true` | Create as draft PR |
| `labels` | string[] | `["nax-generated"]` | Labels to apply |
| `assignee` | string | `undefined` | GitHub assignee (`@me` for self) |
| `template` | string | `"default"` | PR body template name |
| `autoMerge` | boolean | `false` | Enable auto-merge after CI passes |
| `skipIfExists` | boolean | `true` | Skip if a PR already exists for this branch |

### PR Body Template (default)

```markdown
## Feature: {{feature}}

{{description}}

## Stories

| ID | Title | Status |
|:---|:------|:-------|
{{#stories}}
| {{id}} | {{title}} | {{status}} |
{{/stories}}

## Changes

{{diffStat}}

### Commits

{{commitLog}}

## Acceptance Criteria

{{#acceptanceCriteria}}
- [x] {{.}}
{{/acceptanceCriteria}}

## Metrics

- **Duration:** {{duration}}
- **Cost:** ${{cost}}
- **Stories:** {{completed}}/{{total}} completed

---
*Generated by [nax](https://github.com/nathapp-io/nax) v{{version}}*
```

### Prerequisites

- `gh` CLI installed and authenticated (`gh auth status`)
- Git remote points to GitHub

### Behavior

1. **setup()** ‚Äî verify `gh` CLI is installed and authenticated. If not, log warning and disable.
2. **shouldRun(context)** ‚Äî return true if: all stories passed, `gh` is available, no existing PR for branch
3. **execute(context):**
   a. Read PRD from `nax/features/<feature>/prd.json`
   b. Detect current branch and base branch
   c. Collect `git diff --stat` and `git log --oneline` against base
   d. Build PR body from template
   e. Create PR via `gh pr create --body-file`
   f. Apply labels, assignee, draft status
   g. Return `{ success: true, message: "PR #N created", url: "https://..." }`

### Stories

| ID | Title | Complexity |
|:---|:------|:-----------|
| GH-PR-001 | Plugin scaffold ‚Äî setup, teardown, config validation | Simple |
| GH-PR-002 | PR body builder ‚Äî PRD reader, diff collector, template renderer | Medium |
| GH-PR-003 | GitHub PR creation ‚Äî `gh` CLI integration, duplicate detection | Medium |
| GH-PR-004 | Unit + integration tests | Medium |

## Package: `@nathapp/nax-plugin-gitlab-mr`

### Config

```json
{
  "plugins": [
    {
      "module": "@nathapp/nax-plugin-gitlab-mr",
      "config": {
        "targetBranch": "main",
        "draft": true,
        "labels": ["nax-generated"],
        "assignee": "@me",
        "removeSourceBranch": true
      }
    }
  ]
}
```

| Field | Type | Default | Description |
|:------|:-----|:--------|:------------|
| `targetBranch` | string | `"main"` | Target branch for MR |
| `draft` | boolean | `true` | Create as draft MR |
| `labels` | string[] | `["nax-generated"]` | Labels to apply |
| `assignee` | string | `undefined` | GitLab assignee |
| `removeSourceBranch` | boolean | `true` | Delete source branch on merge |
| `template` | string | `"default"` | MR body template name |
| `skipIfExists` | boolean | `true` | Skip if an MR already exists for this branch |
| `useGlab` | boolean | `true` | Use `glab` CLI. If false, use `git push -o` options |

### MR Body Template (default)

Same structure as GitHub template ‚Äî both share the PR body builder module.

### Prerequisites

**Option A (preferred):** `glab` CLI installed and authenticated
**Option B (fallback):** Git remote is GitLab ‚Äî uses `git push` with merge request options:
```bash
git push -o merge_request.create \
         -o merge_request.target=main \
         -o merge_request.title="feat: <title>" \
         -o merge_request.description="<body>" \
         -o merge_request.label="nax-generated" \
         -o merge_request.draft
```

### Behavior

1. **setup()** ‚Äî detect GitLab remote. Try `glab auth status`. If `glab` unavailable, fall back to `git push -o` mode.
2. **shouldRun(context)** ‚Äî return true if: all stories passed, GitLab remote detected
3. **execute(context)** ‚Äî same flow as GitHub plugin, but:
   a. Uses `glab mr create` or `git push -o merge_request.create`
   b. `git push -o` fallback has limited feature support (no assignee, limited labels)
   c. Return `{ success: true, message: "MR !N created", url: "https://..." }`

### Stories

| ID | Title | Complexity |
|:---|:------|:-----------|
| GL-MR-001 | Plugin scaffold ‚Äî setup, teardown, config validation, remote detection | Simple |
| GL-MR-002 | GitLab MR creation ‚Äî `glab` CLI + `git push -o` fallback | Medium |
| GL-MR-003 | Unit + integration tests | Medium |

## Shared Module: `nax-plugin-pr-core`

Both plugins share common logic. Extract into an internal shared module (bundled into each plugin, not published separately):

| Module | Purpose |
|:-------|:--------|
| `pr-body-builder.ts` | Read PRD, collect diff/commits, render template |
| `git-utils.ts` | `getCurrentBranch()`, `getDiffStat()`, `getCommitLog()`, `getRemoteUrl()` |
| `template.ts` | Mustache-style template renderer (simple `{{var}}` + `{{#list}}` loops) |
| `cli-check.ts` | `isInstalled(binary)` ‚Äî check if `gh`/`glab` is available and authenticated |

## Testing Strategy

- **Unit tests:** PR body builder, template renderer, git utils (mock `Bun.spawn`)
- **Integration tests:** Full plugin flow with mocked `gh`/`glab` CLI (inject via `_deps`)
- **No E2E:** Don't actually create PRs in tests ‚Äî mock the CLI calls

## Acceptance Criteria

- [ ] GitHub plugin creates draft PR with correct body when all stories pass
- [ ] GitHub plugin skips when stories failed
- [ ] GitHub plugin skips when PR already exists for branch
- [ ] GitHub plugin logs warning and disables when `gh` CLI missing
- [ ] GitLab plugin creates draft MR via `glab` when available
- [ ] GitLab plugin falls back to `git push -o` when `glab` unavailable
- [ ] GitLab plugin skips when stories failed
- [ ] PR body includes: feature description, story table, diff stat, commit log, cost metrics
- [ ] Both plugins are installable via `npm install` and configurable in `nax/config.json`
